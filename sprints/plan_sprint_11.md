## The priority-generation RULES (architect-decided — the NON-AI template logic, decide-and-log)
The brief's numbered priorities are generated by RULES from real data (NOT AI). Each rule checks a condition + emits a priority if met. Decided rules (ordered by importance):
1. **Market/ladder** — if a market asset hit a ladder rung (finance ladder + market price) → "‹asset› chạm rung ‹N› ‹$price› — cân nhắc DCA ‹$amount›, dry powder ‹X%›." (reuse finance ladder + market triggers.)
2. **Project pattern** — if a project matches build-to-90 (progress≥90 & users==0 & not-abandoned — the pattern-check rule, abandon-orthogonal) OR is idle>7d → "‹project› ‹status› — khớp pattern build-to-90 / đứng ‹N›d. Quyết bỏ/đẩy?"
3. **Claude quota** — if claude usage pct high or remaining low → "Quota còn ‹X›/đốt ‹pct%› — ‹đủ/sắp hết› session. Ưu tiên ‹...›." (reuse claude-usage; if stale, note asOf.)
4. **Finance drift** — if allocation drift > threshold (finance driftAlert) → "Phân bổ lệch ‹channel› ‹drift%› — rebalance?"
5. **Idle/alerts** — recent market alerts or idle-hunter findings → surface the top one.
Cap at ~3-5 priorities (the mock shows 3). Each priority = `{n, text, source, severity}`. If NO conditions met → an honest "Mọi thứ ổn định, không có ưu tiên khẩn." (never fabricate urgency).

**Decided severity + thresholds (decide-and-log, §Assumptions):**
- **market/ladder:** rung HIT → **urgent**; price within ~2% of next rung → **info**; else none.
- **project:** build-to-90 (progress≥90 & users==0 & not-abandoned) → **urgent**; idle>7d & not-abandoned → **warn**.
- **claude quota:** pct≥90 → **urgent**, pct≥75 → **warn**, <75 → none; if claude stale → cap at warn + note asOf.
- **finance drift:** driftAlert (finance's |drift|>5%, server-owned) → **warn**.
- **alerts:** top non-ladder market alert today → **warn** (ladder hits are rule 1, no dup); else none.
- **severity enum:** `Literal["info","warn","urgent"]`.
- **sort:** DESC by severity, tiebreak = rule order (market→project→claude→finance→alerts) for determinism. `n` = 1-based DISPLAY rank after sort.
- **asOf** = min(source asOf) (claude's cache usually oldest); **stale** = any source stale. **summary** = structured {netWorth, projectsActive, claudePct, alertsToday} (FE composes the string). **honest-empty** = priorities:[] + real summary; the "ổn định" text is FE-side.

```
}
```

## Logic/Algorithm (architect-decided — the priority RULES, decide-and-log)
- **generate:** read finance.overview + projects.list + market.get + claude-usage.get + market alerts (the morning-pull pattern). Each source fail-soft (down → warning, skip its rules).
- **priorities:** run the 5 rules (above) — each emits 0-1 priorities. **The rule NUMBER (1-5) is the GENERATION source; the DISPLAY order is by computed SEVERITY** (urgent>warn>info), then source-order as tiebreak. A critical alert outranks a mild drift regardless of which rule produced it (team-lead refinement — the mock's DCA→build90→quota order was that day's severity, not a fixed rule order). Carry `severity` so it's self-describing + FE-sortable. Cap ~5; `n` = 1-based DISPLAY rank (after the severity sort).
- **summary:** netWorth=finance.totalValue, projectsActive=count(health in act/slow & not abandoned), claudePct=claude.pct, alertsToday=count(market alerts today).
- **build-to-90 rule (abandon-orthogonal):** a project priority fires for progress≥90 & users==0 & not-abandoned (NOT health=dead) — same as pattern-check. A health=dead project that's NOT 90%-0-user is NOT a build-to-90 priority.
- **stale:** true if any source's data is stale (e.g. claude cache asOf old) → surface it (don't imply live).
- **no priorities:** honest "Mọi thứ ổn định" — NEVER fabricate urgency to fill the list.
- **NO AI:** all rules are deterministic conditions on real data. No LLM. (The "08:42 · opus" in the mock is the FUTURE Claude-Code-generated brief — this build is template, label it "template" not "opus".)

## Defensive (MANDATORY)
