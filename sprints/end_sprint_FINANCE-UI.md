## Assumptions (user-review)
- **Honest-confidence render = two distinct visuals + a band threshold (display-only).** `confidenceBand` buckets the backend `confidence` (<0.4 thấp/neg · <0.7 vừa/mid · ≥0.7 cao/pos) into a TONE so low trust de-emphasizes — a DISPLAY choice on the backend value, NOT a recomputation. **Why:** the §116 weight≠confidence lock — a thin W must read low-conviction, not a green "go". **How to change:** the thresholds in `confidenceBand` (lib/useDecision.ts).
- **NEUTRAL DOM gate regex** = `buy|sell|should|rebalance|deploy|recommend|ought` (word-boundary), omitting move/must (false-positive-prone). **Why:** catch unambiguous advice imperatives the cockpit might add; guardian/verdict strings are backend-neutral, rendered verbatim. **How to change:** the FORBIDDEN regex in decision.test.tsx if the copy set grows.
- **Per-coin vs channel P&L distinctly surfaced** — per-coin `holding.pnl` (the S3-deepened, basis-aware) vs channel `allocation.pnl` (basisUnknown→null), labeled separately, both null-safe "—". **Why:** the S12A trap — never conflate the two pnl shapes, never fabricate a number for a basis-less coin. **How to change:** N/A (the schema dictates it).
- **crypto-basis form is channel-level; per-coin off-OKX manual basis = a flagged FOLLOW-UP** (small backend endpoint + tiny FE per-row form) — NOT in this sprint.
